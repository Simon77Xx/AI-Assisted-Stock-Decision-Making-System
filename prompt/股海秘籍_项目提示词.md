# 项目任务说明：股海秘籍 - 股票行情数据分析之旅

## 项目背景
这是一个课程项目，最终会向公司同事（非专业炒股的互联网从业者）做展示，同时我希望做完后能自己日常使用。开发者本人具备 Python/pandas 基础，时间有限（1-2周），请优先保证核心闭环可跑通、代码清晰可维护，不要过度设计。

## 技术栈
- 数据获取：akshare（A股数据，使用前复权 qfq）
- 数据处理：pandas
- 回测：自己实现简单的事件驱动回测，不引入 backtrader/vectorbt 等重型框架
- 前端展示：Streamlit
- 数据缓存：本地 parquet 文件

## 核心功能范围（严格按此范围，不要自行扩展）
1. 输入股票代码 + 时间区间，通过 akshare 获取日线数据（前复权），本地缓存避免重复请求
2. 计算双均线（MA5 / MA20）金叉死叉信号
3. 加一个趋势过滤器：用 MA60 判断当前是否处于趋势市，只有判定为趋势市时才启用双均线信号，否则空仓
4. 正确处理信号执行时滞：**T日收盘后生成信号，T+1日才执行交易**，禁止使用未来数据（这是本项目的关键正确性要求，务必用 `shift(1)` 等方式严格隔离信号日和交易日）
5. 回测输出核心指标：累计收益率、最大回撤、夏普比率、胜率，并与"买入持有"基准对比
6. Streamlit 页面：
   - 侧边栏：股票代码输入、时间区间选择、均线参数可调（滑块）
   - 主区域：K线图（叠加MA5/MA20/MA60和买卖点标记）+ 策略收益曲线 vs 买入持有曲线 + 指标卡片展示

## 明确不做的事（避免范围蔓延）
- 不做多因子选股、不做财务数据分析
- 不做滑点/手续费的精细仿真，先做最朴素版本能跑通即可，预留接口后续可加
- 不做用户登录、数据库等工程化内容，本地单用户使用即可
- 不做实盘交易接口对接

## 关键正确性要求（这是本项目最容易出错也最容易被人看穿的地方，请认真处理）
- 严禁 look-ahead bias（未来函数）：任何信号必须基于截至 T 日收盘的信息，执行只能在 T+1 日，代码中需要有清晰注释说明这一点
- 必须使用前复权数据（akshare 的 `adjust="qfq"` 参数），避免除权除息导致的价格断层产生假信号
- 数据获取需要做本地缓存（按股票代码存 parquet），并做基本的异常处理（接口失败时给出清晰报错，不要静默出错）

## 建议的开发顺序（请按此顺序分阶段实现，每阶段完成后我会review再继续）
1. 数据层：akshare 拉取 + 缓存 + 复权处理，先在脚本/notebook里验证拿到的数据是对的
2. 策略与回测层：双均线信号 + 趋势过滤 + 正确的 T+1 执行 + 核心指标计算，先用单只股票验证回测逻辑无误（特别检查没有未来函数）
3. Streamlit 前端：先做最简版本跑通交互，再迭代可视化效果

## 代码风格要求
- 函数职责单一，数据获取、信号计算、回测、绘图分别成独立函数/模块，方便我后续替换策略或加新功能
- 关键逻辑（尤其是 shift(1) 那一行）必须有注释解释原因
- 不要引入不必要的依赖
